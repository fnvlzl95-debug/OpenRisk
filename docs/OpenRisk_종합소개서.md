# 1. 오픈리스크란?

## 1.1 한 줄 정의

> **"창업 실패를 막는 공공데이터 기반 상권 리스크 분석 서비스"**

## 1.2 왜 만들었나?

한국에서 자영업 창업은 흔하지만, **창업 후 3년 내 폐업률이 60%를 넘습니다.**

대부분의 창업자들은:
- "여기 사람 많아 보이니까 괜찮겠지"
- "내 음식이 맛있으니까 잘 되겠지"
- "부동산에서 좋다고 했으니까"

이런 **감에 의존한 판단**으로 창업합니다.

오픈리스크는 이런 사람들에게 **숫자로 된 경고**를 보여줍니다.
- 반경 500m에 같은 업종이 몇 개 있는지
- 이 지역 폐업률이 얼마나 높은지
- 임대료 대비 유동인구가 적절한지

**"창업을 말리는 AI"** - 이것이 우리의 포지션입니다.

## 1.3 핵심 가치

| 기존 상권 분석 | 오픈리스크 |
|---------------|-----------|
| 낙관적 ("여기 좋아요!") | 비관적 ("여기 위험해요") |
| 감에 의존 | 데이터 기반 |
| 유료/복잡 | 무료/간단 |
| 전문가용 | 일반인용 |

## 1.4 타겟 사용자

1. **예비 창업자**: 처음 창업하려는 사람
2. **기존 자영업자**: 2호점/이전 검토 중인 사람
3. **프랜차이즈 본사**: 가맹점 입지 분석
4. **부동산 중개인**: 상가 임대 상담 자료

## 1.5 서비스 범위

- **지역**: 서울 25개구 + 경기 31개 시/군 + 인천 8개 구/군 = **총 64개 지역**
- **업종**: 18개 카테고리
- **분석 반경**: 선택 위치 기준 500m

# 2. 사용 중인 데이터와 지표

## 2.1 데이터 소스 (어디서 가져오나?)

### 2.1.1 점포 데이터 (핵심)

| 항목 | 내용 |
|------|------|
| **출처** | 소상공인시장진흥공단 |
| **이름** | 상가(상권)정보 |
| **갱신주기** | 매월 |
| **현재 버전** | 2025년 10월 |
| **데이터 크기** | 서울 279MB + 경기 339MB + 인천 69MB |
| **총 레코드** | 약 450만개 점포 |

**포함 정보**:
- 상호명
- 업종 분류 (소분류까지)
- 주소 (도로명, 지번)
- 좌표 (위도, 경도)

**우리가 하는 일**:
1. CSV 파일을 파싱
2. 업종을 우리 18개 카테고리로 매핑
3. H3 육각형 그리드(반경 약 175m)로 집계
4. Supabase 데이터베이스에 저장

### 2.1.2 지하철 데이터

| 항목 | 내용 |
|------|------|
| **출처** | 서울교통공사, 인천교통공사 |
| **내용** | 역별 일평균 승하차 인원 |
| **현재 버전** | 2023년 |
| **역 수** | 서울 1~9호선 + 인천 |

**포함 정보**:
- 역 이름, 노선
- 좌표
- 일평균 승하차 (평일/주말 구분)
- 시간대별 승하차 (06시~23시)

**우리가 사용하는 방식**:
- 선택한 위치에서 가장 가까운 역 찾기
- 역의 승하차 데이터로 유동인구 지수 계산
- 시간대별 패턴 분석 (아침/낮/저녁)

### 2.1.3 버스 데이터

| 항목 | 내용 |
|------|------|
| **출처** | 전국 버스정보시스템 |
| **내용** | 정류장별 승하차 인원 |
| **현재 버전** | 서울 2022년, 인천 2025년 |

**우리가 사용하는 방식**:
- 반경 300m 내 버스 정류장 찾기
- 정류장별 승하차로 유동인구 보정

### 2.1.4 임대료 데이터

| 항목 | 내용 |
|------|------|
| **출처** | 한국부동산원 + 자체 수집 |
| **단위** | 구별 평균 (만원/평) |
| **현재 버전** | 2025년 Q3 |

**한계**:
- 구 단위 평균이라 같은 구 내에서도 편차가 큼
- 강남구 평균 68만원이지만 신사동과 개포동은 다름

### 2.1.5 외부 API

**카카오 API** (실시간 호출):
- 역지오코딩: 좌표 → 주소 변환
- 장소 검색: 스타벅스, 대형마트, 백화점 위치

**OpenAI API**:
- GPT-4.1-mini 모델
- AI 분석 요약 생성

## 2.2 7대 핵심 지표

### 지표 1: 경쟁 밀도 (Competition)

**측정 방법**: 반경 500m 내 동종 업종 점포 수

| 수준 | 동종 업종 수 | 의미 |
|------|-------------|------|
| 여유 | 0~5개 | 경쟁 적음 (단, 수요도 없을 수 있음) |
| 보통 | 6~10개 | 적당한 경쟁 |
| 포화 | 11~15개 | 경쟁 치열 |
| 과포화 | 16개 이상 | 진입 비추천 |

**예시**:
- 홍대입구역 500m 내 카페: 158개 → **과포화**
- 외곽 주거지역 카페: 3개 → **여유** (단, 수요 검증 필요)

---

### 지표 2: 유동인구 (Traffic)

**측정 방법**: 지하철 + 버스 승하차 기반 추정 지수

**계산 공식**:
```
유동인구 지수 = (지하철 컴포넌트 + 버스 컴포넌트) × POI 보정
```

- **지하철 컴포넌트** (0~50점): 가까운 역의 일평균 승하차 기반
- **버스 컴포넌트** (0~30점): 주변 버스정류장 승하차 기반
- **POI 보정** (0.8~1.5): 주변 상가 밀도로 보정

| 레벨 | 지수 | 설명 |
|------|------|------|
| 매우 낮음 | 0~10 | 주거 밀집, 워킹 고객 거의 없음 |
| 낮음 | 11~25 | 주거-상업 혼합 |
| 보통 | 26~50 | 일반 상업지역 |
| 높음 | 51~80 | 역세권, 번화가 |
| 매우 높음 | 81~100 | 핵심 상권 (강남역, 홍대 등) |

**시간대 분석**:
- 아침 (06~11시): 출근 시간대
- 낮 (11~17시): 점심/오후
- 저녁 (17~23시): 퇴근/야간

**왜 중요한가?**:
- 유동인구 ≠ 내 가게 고객
- 지나가는 사람이 많아도 들어올지는 다른 문제
- 시간대별 분석으로 "언제 장사가 될지" 파악

---

### 지표 3: 임대료 (Cost)

**측정 방법**: 해당 구의 평균 상가 임대료 (만원/평)

| 레벨 | 임대료 | 예시 |
|------|--------|------|
| 낮음 | ~20만원/평 | 외곽 주거지역 |
| 보통 | 21~35만원/평 | 일반 상업지역 |
| 높음 | 36만원~/평 | 강남, 홍대 등 핵심 상권 |

**왜 중요한가?**:
- 월세가 높으면 손익분기점도 높아짐
- 10평 카페, 월세 300만원이면 하루 10만원은 벌어야 임대료만 나옴
- 유동인구 대비 임대료가 너무 높으면 위험

---

### 지표 4: 생존율/폐업률 (Survival)

**측정 방법**: 해당 지역의 점포 변동 분석

| 항목 | 설명 |
|------|------|
| 폐업률 | (폐업 점포 / 이전 기간 점포) × 100 |
| 개업률 | (신규 점포 / 이전 기간 점포) × 100 |
| 순증감 | 개업 - 폐업 |
| 트렌드 | 성장 / 안정 / 감소 |

**위험도 분류**:
| 위험도 | 폐업률 | 의미 |
|--------|--------|------|
| 안정 | ~5% | 안정적인 상권 |
| 보통 | 6~10% | 일반적인 수준 |
| 주의 | 11%~ | 점포 이탈이 많은 지역 |

**왜 중요한가?**:
- 폐업률이 높다 = 이미 많은 사람이 실패함
- "왜 다른 사람들은 못 버텼을까?"를 생각해야 함

---

### 지표 5: 앵커 시설 (Anchor)

**앵커 시설이란?**: 사람을 끌어모으는 큰 시설

| 시설 | 탐색 반경 | 의미 |
|------|----------|------|
| 지하철역 | 1km | 출퇴근 유동인구 |
| 스타벅스 | 1km | 상권 활성도 지표 |
| 대형마트 | 2km | 생활권 중심지 |
| 백화점 | 2km | 고급 상권 |

**왜 중요한가?**:
- 역에서 5분 거리 vs 15분 거리 = 유입량 차이 큼
- 스타벅스가 있다 = "여기 돈 되겠다"고 판단한 것
- 앵커가 없으면 자체 집객력이 있어야 함

---

### 지표 6: 시간대 패턴 (Time Pattern)

**측정 방법**: 지하철 시간대별 승하차 비율

| 패턴 | 설명 | 적합 업종 |
|------|------|----------|
| 아침형 | 아침 비중 40%+ | 카페, 베이커리 |
| 점심형 | 낮 비중 40%+ | 식당, 테이크아웃 |
| 저녁형 | 저녁 비중 45%+ | 술집, 고깃집 |

**왜 중요한가?**:
- 저녁형 상권에서 아침 장사 = 공백 시간 많음
- 주말 비중 높은 곳 = 평일 매출 방어 전략 필요

---

### 지표 7: 상권 유형 (Area Type)

**4가지 유형**:

| 유형 | 코드 | 특징 |
|------|------|------|
| 주거형 | A_주거 | 주거 밀집, 생활 밀착형, 단골 중심 |
| 혼합형 | B_혼합 | 직장+주거, 시간대별 고객층 다름 |
| 상업형 | C_상업 | 유동인구↑, 경쟁↑, 임대료↑ |
| 특수형 | D_특수 | 관광지/이벤트, 시즌 의존 |

**유형 판별 기준**:
- A_주거: 점포 밀도 낮음 OR 주거형 점포 40%+
- C_상업: 유동지수 30+ AND 상업 점포 60%+
- D_특수: 주말 비율 50%+ OR (역 300m + 상업 70%+)
- B_혼합: 위의 어느 것도 아님

## 2.3 종합 리스크 점수 (0~100)

**계산 방식**: 업종별 가중치 적용

**예시 - 카페**:
| 지표 | 가중치 | 설명 |
|------|--------|------|
| 경쟁 | 35% | 카페는 경쟁이 가장 중요 |
| 임대료 | 20% | 객단가 낮아 임대료 부담 큼 |
| 폐업률 | 20% | 폐업 많은 업종 |
| 유동인구 | 20% | 워킹 고객 필요 |
| 앵커 | 5% | 보조 지표 |

**점수 해석**:
| 점수 | 레벨 | 의미 |
|------|------|------|
| 0~29 | 낮음 | 상대적으로 안전 |
| 30~49 | 보통 | 주의 필요 |
| 50~69 | 높음 | 신중한 검토 필요 |
| 70~100 | 매우 높음 | 창업 비추천 |

# 3. 제공하는 모든 기능

## 3.1 사용자 기능

### 3.1.1 홈 화면

**두 가지 디자인 (스킨)**:
1. **Glasmorphic (스킨 A)**: 다크 테마, 미래적 느낌
2. **Editorial (스킨 B)**: 신문 스타일, 전문적 느낌

**기능**:
- 업종 선택 (18개 중 택1)
- 주소/지역 검색 (자동완성)
- 빠른 검색 태그 (홍대입구역, 강남역 등)

### 3.1.2 검색 기능

**주소 검색**:
- 실시간 자동완성 (200ms 딜레이)
- 카카오 API 연동
- 검색 결과에서 선택하면 좌표 자동 입력

**업종 선택**:
- 그룹별 아코디언 형식
- 선택 필수 (안 하면 경고)

### 3.1.3 분석 결과 화면

**상단 정보**:
- 위치 주소
- 선택한 업종
- 상권 유형 (주거형/혼합형/상업형/특수형)
- 종합 리스크 점수 (0~100)

**핵심 지표 카드**:
- 경쟁 밀도: 동종 업종 수 + 레벨
- 유동인구: 지수 + 5단계 레벨
- 임대료: 평당 가격 + 3단계 레벨

**시간대별 분석**:
- 아침/낮/저녁 비율 차트
- 피크 시간대 표시
- 주말 비율

**상권 트렌드**:
- 점포 증가세/보합/감소세
- 폐업률
- 순증감

**앵커 시설 정보**:
- 가장 가까운 지하철역 + 거리
- 스타벅스 개수 + 거리
- 대형마트/백화점 정보

**리스크 카드** (핵심 기능):
- Top 3 리스크 요약
- 레드플래그 형식
- 현장 확인 질문 포함

**데이터 신뢰도 표시**:
- 점포 데이터 수집 시기
- 커버리지 레벨

### 3.1.4 AI 분석 기능

**트리거**: 우측 하단 "AI 분석" 버튼

**AI가 제공하는 내용**:
1. **헤드라인**: 핵심 경고 한 줄
2. **리스크 분석**: 3문단 상세 분석
3. **실패 시나리오**: 예상 실패 경로
4. **현장 체크리스트**: 직접 확인할 것 3~5개
5. **최종 경고**: 마지막 권고

**AI의 포지션**: "창업을 말리는 AI"
- 긍정적 표현 금지

# 4. 데이터 정확도와 개선 방안

## 4.1 현재 정확도 평가

### 4.1.1 점포 데이터 (정확도: ★★★★☆)

**강점**:
- 공공데이터라 신뢰도 높음
- 매월 갱신
- 전국 450만개 점포 커버

**약점**:
- 업종 분류 오류 (예: "카페"인데 "다방"으로 등록)
- 폐업 반영 지연 (1~2개월)
- 신규 점포 반영 지연

**개선 방안**:
1. **크롤링 보완**: 네이버 플레이스, 카카오맵 크롤링으로 교차 검증
2. **사용자 신고**: "이 정보가 틀려요" 버튼 추가
3. **AI 분류 보정**: 상호명으로 업종 재분류

### 4.1.2 유동인구 데이터 (정확도: ★★★☆☆)

**강점**:
- 지하철 데이터는 정확함
- 시간대별 분석 가능

**약점**:
- "승하차 인원 ≠ 지나가는 사람"
- 역이 없는 지역은 추정치
- 버스 데이터 오래됨 (서울 2022년)

**개선 방안**:
1. **KT/SKT 유동인구 데이터**: 유료지만 정확함
2. **CCTV 분석**: 카운팅 기술 적용
3. **버스 데이터 갱신**: 경기/인천 최신 데이터 확보

### 4.1.3 임대료 데이터 (정확도: ★★☆☆☆)

**강점**:
- 구 단위 트렌드는 파악 가능

**약점**:
- 구 단위 평균이라 정확도 낮음
- 같은 강남구라도 신사동 vs 개포동 = 2배 차이
- 권리금 미반영

**개선 방안**:
1. **동 단위 세분화**: 법정동별 임대료 수집
2. **건물별 데이터**: 등기부등본/임대차 계약 데이터
3. **부동산 플랫폼 연동**: 직방, 다방 API

### 4.1.4 폐업률 데이터 (정확도: ★★★☆☆)

**강점**:
- 점포 변동으로 계산 가능

**약점**:
- 10개월치 데이터만 있음
- 업종별 폐업률은 추정

**개선 방안**:
1. **시계열 확보**: 12개월 이상 데이터 축적
2. **통계청 연동**: 업종별 생존율 통계
3. **폐업 사유 수집**: 왜 폐업했는지 분석

### 4.1.5 앵커 시설 (정확도: ★★★★☆)

**강점**:
- 카카오 API 실시간 조회
- 거리 계산 정확

**약점**:
- 카카오에 등록 안 된 시설 누락
- 시설의 "규모"는 미반영

**개선 방안**:
1. **구글 Places API 병행**: 더 많은 POI 확보
2. **시설 규모 반영**: 스타벅스 리저브 vs 일반 매장

## 4.2 데이터 품질 관리

### 현재 하고 있는 것

1. **데이터 커버리지 표시**: 사용자에게 불확실성 공개
   - High: 점포 데이터 5개+ 셀 + 유동 데이터 3개+ 셀
   - Medium: 부분적 데이터
   - Low: 데이터 부족

2. **Admin API**: 데이터 상태 확인 및 패치
   - 누락된 카테고리 0으로 초기화
   - 비정상 값 (weekend_ratio > 1) 정정

3. **갱신 주기 명시**: "2025년 10월 기준"

### 더 해야 할 것

1. **자동 검증 파이프라인**
2. **이상치 알림 시스템**
3. **사용자 피드백 루프**
