> 예산: 월 0~30만 원
> 운영 방식: Founder-led 텍스트/커뮤니티 직접 운영, 얼굴 노출 없음

## 1. 현재 상황 진단

- 최근 7일 방문자: `12`
- 최근 7일 페이지뷰: `42`
- 최근 7일 이탈률: `58%`
- 주요 방문 경로는 `/home-b` 중심
- 외부 유입은 `fmkorea.com 1`, `google.com 1` 수준
- 현재 병목은 전환보다 유입 부족

### 진단 요약

지금은 랜딩 최적화보다 먼저 사람을 데려오는 일이 더 중요하다.  
트래픽 규모가 너무 작아서 이탈률이나 미세한 UI 차이로 성과를 해석하기 어렵다.  
우선순위는 `검색에 걸릴 자산 증가`, `Founder-led 배포`, `전환 이벤트 측정`이다.

### 이미 완료된 기반 작업

- `robots.txt` 설정 완료
- `sitemap.xml` 제출 완료
- Google Search Console 등록 완료
- Naver Search Advisor 등록 완료
- 사이트맵 제출 및 주요 URL 색인 점검 완료

## 2. 목표와 KPI

### 8주 목표

- 주간 방문자: `80~150`
- 주간 분석 시작 수: `20+`
- 외부 유입 채널: `5개+`

### KPI 우선순위

1. `analysis_loaded`
2. 외부 referrer 수
3. 주간 방문자
4. 이탈률

### 운영 원칙

- 방문자 수보다 실제 분석 시작 수를 더 중요하게 본다.
- 반응 없는 채널은 빨리 끊고, 반응 있는 채널만 반복한다.
- 추상적인 브랜딩보다 검색 자산과 실제 배포량을 우선한다.

## 3. 핵심 전략

### 3.1 홈 메시지 단순화

- 첫 화면 메시지를 `3분 창업 위험도 진단` 중심으로 정리한다.
- 메인 CTA는 `무료 진단 받기` 하나만 남긴다.
- 목적은 3초 안에 서비스 이해를 끝내는 것이다.

### 3.2 검색 자산 확장

- 인덱서블 페이지 수를 늘려야 한다.
- 정보톡과 가이드형 콘텐츠를 주 4건 수준으로 누적한다.
- 업종형, 입지형, 비교형, 체크리스트형 키워드를 중심으로 확장한다.

### 3.3 Founder-led 커뮤니티 배포

- 광고보다 직접 배포가 우선이다.
- 네이버 블로그, 네이버 카페, FMKorea, 자사 정보톡 4개 채널만 집중한다.
- 링크만 던지지 않고 사례 중심으로 먼저 가치를 제공한다.

### 3.4 이벤트 기반 측정 정비

- 지금은 페이지 방문만 보여서 퍼널이 보이지 않는다.
- 홈 진입, CTA 클릭, 검색 제출, 분석 완료, 공유, PDF 저장, 피드백까지 이벤트를 잡아야 한다.
- 마케팅 효율은 채널별 방문이 아니라 이벤트별 전환으로 판단한다.

## 4. 1주차 즉시 실행안

### Day 1

- 콘텐츠: 기존 정보톡 글 5개를 기준으로 재가공 우선순위를 정한다.
- 배포: 네이버 블로그 1건 발행 준비, FMKorea용 글감 1건 정리.
- 측정: `home_view`, `cta_click`, `query_submit`, `analysis_loaded` 이벤트 정의.

### Day 2

- 콘텐츠: 네이버 블로그용 검색형 글 1건 발행.
- 배포: FMKorea 1건 발행, 관련 글 댓글 10개 참여.
- 측정: referrer와 UTM 저장 방식 점검.

### Day 3

- 콘텐츠: 정보톡 후속글 1건 발행.
- 배포: 네이버 카페 1곳에 질문형 글 또는 댓글 참여.
- 측정: 홈 방문 대비 분석 시작 수 첫 집계.

### Day 4

- 콘텐츠: 네이버 블로그 1건 추가 발행.
- 배포: FMKorea 1건 추가 발행, 자사 정보톡에서 댓글 유도.
- 측정: 유입 채널별 `analysis_loaded` 비율 확인.

### Day 5

- 콘텐츠: 샘플 분석 사례 2건 정리.
- 배포: 커뮤니티 댓글 10개, 블로그 글 1건 재배포.
- 측정: 가장 클릭이 나는 주제 2개 선정.

### Day 6

- 콘텐츠: 비교형 글 1건 작성.
- 배포: 네이버 카페 또는 지역 커뮤니티 1곳 추가 테스트.
- 측정: referrer 수, 방문 수, 분석 시작 수를 7일 누적으로 정리.

### Day 7

- 콘텐츠: 1주차 반응이 좋았던 주제를 다음 주 캘린더로 확정.
- 배포: 반응 없던 채널 1개를 중단하고 반응 있는 채널만 남긴다.
- 측정: 주간 리뷰를 통해 Week 2 집중 주제를 결정한다.

## 6. 채널별 운영 원칙

### 네이버 블로그

- 검색형 제목으로 쓴다.
- 한 글에 한 키워드 축만 잡는다.
- 서비스 소개보다 사례와 체크리스트 중심으로 쓴다.

### 네이버 카페

- 가입 후 바로 홍보하지 않는다.
- 질문 답변과 댓글 참여를 먼저 한다.
- 글 본문 80%는 정보, 마지막 20%만 링크 또는 CTA로 남긴다.

### FMKorea

- 질문형, 경험형, 비교형 제목이 유리하다.
- 서비스 자랑보다 “이 상권 왜 위험해 보이냐” 식의 사례형으로 접근한다.
- 반응이 없으면 같은 톤을 반복하지 않는다.

### 자사 정보톡

- 검색형 글과 토론형 글을 섞는다.
- 단순 게시보다 후속 댓글이 달릴 수 있는 질문 구조를 만든다.
- 검색 자산이자 신뢰 자산으로 계속 누적한다.

## 7. 콘텐츠 운영 캘린더

### 우선 주제 16개

1. 상권분석 초보 체크리스트
2. 창업 입지 보는 법
3. 무인카페 창업 위험도
4. 개인카페 상권 체크
5. 고깃집 입지 체크
6. 분식집 창업 상권
7. 배달전문점 입지
8. 신규 아파트 상가
9. 오피스 상권 vs 주거 상권
10. 1층 vs 2층/지하
11. 통과형 vs 체류형 상권
12. 역세권 함정
13. 권리금 전 체크리스트
14. 월세 비율 계산법
15. 폐업률 높은 상권 신호
16. 프랜차이즈 vs 개인창업 상권 선택

### 운영 방식

- 주 4건 생산을 기본으로 한다.
- 검색형 2건, 비교형 1건, 체크리스트형 1건으로 구성한다.
- 가장 반응 있는 주제를 다음 주에도 재활용한다.

## 8. 측정 항목과 주간 리뷰 방식

### 필수 이벤트

- `home_view`
- `cta_click`
- `query_submit`
- `analysis_loaded`
- `share_click`
- `pdf_download`
- `feedback_submit`

### 저장 파라미터

- `page_path`
- `referrer`
- `utm_source`
- `utm_medium`
- `utm_campaign`
- `landing_path`

### 주간 리뷰 방식

- 매주 1회 최근 7일 데이터를 리뷰한다.
- 채널별 방문 수보다 채널별 `analysis_loaded`를 먼저 본다.
- 반응이 없던 주제는 바로 제외한다.
- 반응이 있었던 주제는 제목과 각도만 바꿔 반복한다.

지금 단계에서는 채널을 넓히는 것보다 반응이 있는 채널에 집중하는 편이 훨씬 효율적이다.  
제품을 예쁘게 설명하는 일보다 실제 검색 자산과 사례를 늘리는 일이 먼저다.  
성과가 없는 활동은 빠르게 끊고, 신호가 있는 활동만 반복해야 한다.
