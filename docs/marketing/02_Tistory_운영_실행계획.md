# OpenRisk Tistory 운영 실행 계획

> 작성일: 2026-03-28
> 채널: Tistory
> 목적: 검색형 롱테일 유입 확보와 OpenRisk 분석 시작 수 증가
> 운영 방식: 직접 작성, 수동 로그인 세션 재사용, 브라우저 자동화 보조

---

## 1. 채널 역할

- Tistory는 즉시 반응형 커뮤니티보다 검색 누적형 채널로 본다.
- 네이버 블로그보다 자유로운 본문 구성과 외부 링크 활용이 쉬워서 가이드형 글 축적에 유리하다.
- OpenRisk의 첫 번째 역할은 "도구 홍보"가 아니라 "창업 입지 판단 기준을 설명하는 검색 자산"을 만드는 것이다.

## 2. 운영 전제

- 티스토리 Open API는 공식 공지 기준으로 2023-12-22에 종료 안내가 올라왔고, 2024년 2월 말까지 순차 종료됐다.
- 따라서 자동 발행은 API 연동이 아니라 브라우저 자동화 기준으로 운영한다.
- 발행 자동화는 편집기 UI가 바뀌면 셀렉터 조정이 필요할 수 있다.

공식 공지:
- `https://notice.tistory.com/2664`

## 3. 목표

- 첫 4주 동안 검색형 글 12건 이상 누적
- 주 3건 발행 루틴 정착
- 글당 1개 이상의 명확한 검색 의도 유지
- Tistory 유입에서 `analysis_loaded` 전환이 발생하는 글 3개 이상 발굴

## 4. 발행 원칙

- 제목은 한 글에 한 검색 의도만 담는다.
- 도입부 3문장 안에 "누가", "무엇을", "왜 봐야 하는지"를 끝낸다.
- 본문은 체크리스트와 비교 기준 위주로 쓰고, 사례를 한 번 섞는다.
- CTA는 마지막 1문장만 둔다.
- 첫 문단부터 서비스 소개를 시작하지 않는다.

## 5. 추천 카테고리 구조

- `상권분석`
- `창업입지`
- `업종별체크`

카테고리는 너무 잘게 나누지 않는다. 처음 20건 전까지는 2~3개만 유지한다.

## 6. 태그 운영 원칙

- 글마다 4~6개만 사용한다.
- 같은 뜻의 중복 태그는 줄인다.
- 광범위 태그 2개, 구체 태그 2개, 문제 해결형 태그 1~2개 조합으로 쓴다.

예시:
- `상권분석`, `창업입지`, `배달전문점창업`, `오피스상권`, `주거상권`, `유동인구`

## 7. 4주 발행 루틴

### Week 1

- 기존 정보톡 5건 중 검색 의도가 강한 3건을 티스토리형으로 재가공한다.
- 본문 길이는 1,200~1,800자 범위로 맞춘다.
- 발행 후 FMKorea 또는 카페에 직접 링크를 뿌리지 말고, 글 요약본을 따로 작성한다.

### Week 2

- 업종형 글 2건, 비교형 글 1건 발행
- 기존 글 중 클릭이 있는 제목 패턴 1개 재사용
- OpenRisk 샘플 결과 1건을 사례 문단으로 추가

### Week 3

- 신규 아파트 상가, 1층 vs 2층/지하, 역세권 함정 같은 입지형 글 3건 발행
- 글 하단 CTA 문구를 통일해 분석 시작 유도
- 반응 없는 태그 조합 정리

### Week 4

- 유입이 생긴 주제만 다시 확장
- "업종 + 입지 + 판단 기준" 조합으로 후속 글 3건 발행
- 글별 유입과 분석 시작 수를 기준으로 다음 달 주제군 확정

## 8. 원고 제작 방식

- 1차 초안은 정보톡 원고를 기반으로 만든다.
- 2차 편집에서 제목을 검색형으로 다시 잡는다.
- 3차 편집에서 도입부 3문장과 CTA 1문장을 손본다.
- 발행 전에 태그 4~6개, 대표 카테고리 1개만 붙인다.

## 9. 발행 자동화 운영 방식

- 스크립트 파일: `scripts/tistory-publish.ts`
- 기본 모드는 `draft-first`다.
- 첫 실행에서는 카카오 로그인 세션을 만들기 위해 브라우저가 열린다.
- 로그인 이후에는 `TISTORY_USER_DATA_DIR`에 세션이 저장되어 반복 로그인 없이 재사용한다.
- UI가 바뀌면 제목 입력, 본문 입력, 발행 버튼 셀렉터를 먼저 점검한다.

기본 명령:

```bash
npm run tistory:publish -- --file docs/marketing/tistory/01_배달전문점_창업_상권분석.md
```

로그인 세션만 먼저 만들기:

```bash
npm run tistory:login
```

폴더 전체를 임시저장:

```bash
npm run tistory:draft:all
```

폴더 전체를 순서대로 발행:

```bash
npm run tistory:publish:all
```

바로 발행:

```bash
npm run tistory:publish -- --file docs/marketing/tistory/01_배달전문점_창업_상권분석.md --publish
```

권장 환경변수:

```env
TISTORY_BLOG_NAME=your-blog-name
TISTORY_BLOG_URL=https://your-blog-name.tistory.com
TISTORY_USER_DATA_DIR=.tistory-session
TISTORY_VISIBILITY=public
```

## 10. 주간 확인 항목

- 어떤 제목 패턴에서 유입이 생겼는가
- 어떤 글이 `analysis_loaded`로 이어졌는가
- 어떤 태그 조합이 반복 노출되는가
- 검색 노출은 생기는데 클릭이 없는 글은 무엇인가

## 11. 하지 말아야 할 것

- 티스토리 글 첫 단락에서 바로 OpenRisk를 홍보하는 것
- 한 글에 키워드 두세 개를 동시에 억지로 넣는 것
- 하루에 여러 글을 몰아서 발행하고 방치하는 것
- API 기반 자동 발행이 아직 된다고 가정하는 것
- 티스토리 글을 커뮤니티에 그대로 링크만 던지는 것
