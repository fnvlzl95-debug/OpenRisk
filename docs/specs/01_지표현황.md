## 2. 경쟁 밀도 (Competition)

### 2.1 정의
- **동종 업종 점포 수**: 반경 500m(H3 resolution 8) 내 같은 업종 점포
- **전체 점포 수**: 모든 업종 합계

### 2.2 레벨 분류

| 동종 업종 수 | 레벨 | 설명 |
|-------------|------|------|
| 0~5개 | `low` | 경쟁 적음 |
| 6~15개 | `medium` | 보통 |
| 16개 이상 | `high` | 과포화 |

### 2.3 점수 정규화 (0~100)

```
- 5개 이하: 0~30점 (선형)
- 6~15개: 30~100점 (선형)
- 15개 이상: 100점 고정
```

### 2.4 데이터 구조

```typescript
interface CompetitionMetrics {
  total: number              // 반경 내 전체 점포
  sameCategory: number       // 동종 업종
  density: number            // 밀도 (0~1)
  densityLevel: 'low' | 'medium' | 'high'
  nearestCompetitor?: {
    name: string
    distance: number
  }
}
```

## 3. 유동인구 (Traffic)

### 3.1 정의
지하철 승하차 + 버스 승하차 데이터를 H3 그리드에 맵핑한 추정치

### 3.2 구성 요소

| 구성 | 가중치 | 설명 |
|------|--------|------|
| 지하철 승하차 | 70% | 역 500m 내 승하차 인원 |
| 버스 승하차 | 30% | 정류장 300m 내 승하차 인원 |
| POI 보정 | ×1.0~1.3 | 상업시설 밀집도 보정 |

### 3.3 레벨 분류 (5단계, 백분위 기반)

> 2026.01 개편: 실측 분포 기반 (평균 6, 중앙값 3, 최대 70)

| 지수 | 레벨 | 백분위 | 설명 |
|------|------|--------|------|
| ≤1 | `very_low` | 하위 25% | 거의 유동인구 없음 |
| 2~6 | `low` | 25~75% | 적음 (일반 주거지) |
| 7~13 | `medium` | 75~90% | 보통 (소규모 상권) |
| 14~40 | `high` | 90~99% | 많음 (역세권/상업지) |
| >40 | `very_high` | 상위 1% | 매우 많음 (강남/명동/가산급) |

### 3.4 시간대 패턴

```typescript
interface TimePattern {
  morning: number   // 06-11시 비중 (%)
  day: number       // 11-17시 비중 (%)
  night: number     // 17-23시 비중 (%)
}
```

- **peakTime**: 가장 높은 비중의 시간대
- **weekendRatio**: 주말/평일 비율 (1.0 = 동일)

### 3.5 데이터 구조

```typescript
interface TrafficMetrics {
  index: number              // 유동인구 지수 (0~100)
  level: TrafficLevel
  levelLabel: string
  peakTime: 'morning' | 'day' | 'night'
  weekendRatio: number
  timePattern: TimePattern
}
```

## 4. 임대료 (Cost)

### 4.1 정의
- 구/동 단위 평균 임대료 (만원/평)
- 데이터: `commercial_rent_2025Q3.xlsx`

### 4.2 레벨 분류

| 평당 월세 | 레벨 | 설명 |
|----------|------|------|
| ~80만원 | `low` | 저렴 |
| 81~150만원 | `medium` | 보통 |
| 151만원~ | `high` | 고가 |

### 4.3 점수 정규화 (0~100)

```
- 15만원/평 이하: 0~30점
- 16~40만원/평: 30~100점
- 40만원/평 이상: 100점 고정
```

### 4.4 데이터 구조

```typescript
interface CostMetrics {
  avgRent: number            // 평균 임대료 (만원/평)
  level: 'low' | 'medium' | 'high'
  districtAvg?: number       // 해당 구 평균
}
```

## 5. 생존율 (Survival)

### 5.1 정의
- **폐업률**: (폐업 점포 / 이전 기간 점포) × 100
- **개업률**: (신규 개업 / 현재 점포) × 100
- **순증감**: 개업 - 폐업

### 5.2 리스크 레벨

| 폐업률 | 리스크 | 설명 |
|--------|--------|------|
| ~5% | `low` | 안정 |
| 6~10% | `medium` | 보통 |
| 11%~ | `high` | 주의 |

### 5.3 트렌드 분류

| 순증감 | 트렌드 | 라벨 |
|--------|--------|------|
| +3 이상 | `growing` | 점포 증가세 |
| -2 ~ +2 | `stable` | 보합세 |
| -3 이하 | `shrinking` | 점포 감소세 |

### 5.4 데이터 구조

```typescript
interface SurvivalMetrics {
  closureRate: number        // 폐업률 (%)
  openingRate: number        // 개업률 (%)
  netChange: number          // 순증감
  risk: 'low' | 'medium' | 'high'
  trend: 'growing' | 'stable' | 'shrinking'
  trendLabel: string
  summary: string
}
```

## 7. 상권 유형 (Area Type)

### 7.1 분류 기준

| 유형 | 코드 | 조건 |
|------|------|------|
| 주거형 | `A_주거` | 점포 20개 미만 또는 생활서비스 비율 40%+ |
| 혼합형 | `B_혼합` | 기본값 (주거+상업 균형) |
| 상업형 | `C_상업` | 유동 지수 30+ AND 상업 점포 60%+ |
| 특수형 | `D_특수` | 주말 비율 50%+ 또는 역세권+상업 밀집 |

### 7.2 리스크 패널티

| 유형 | 패널티 | 이유 |
|------|--------|------|
| 주거형 | 0점 | 안정적 배후 수요 |
| 혼합형 | +3점 | 약간의 경쟁 |
| 상업형 | +10점 | 높은 경쟁/임대료 |
| 특수형 | +15점 | 시즌/주말 의존 |

## 9. 리스크 카드 시스템

### 9.1 정의
조건 기반으로 자동 생성되는 경고 카드

### 9.2 심각도 분류

| 심각도 | 색상 | 조건 예시 |
|--------|------|----------|
| `critical` | 빨강 | 3중 리스크, 상권 축소 |
| `warning` | 주황 | 경쟁 과밀, 임대료 높음 |
| `caution` | 노랑 | 시간대 편중, 역세권 아님 |

### 9.3 주요 카드 예시

| ID | 플래그 | 조건 |
|----|--------|------|
| `triple_risk` | 3중 리스크 | 경쟁↑ + 임대료↑ + 유동↓ |
| `comp_high_traffic_low` | 파이 쪼개기 | 동종 10개+ AND 유동 낮음 |
| `survival_critical` | 상권 축소 | 폐업률 높음 AND 순감소 |
| `cost_high_comp_high` | 이중 압박 | 임대료 높음 AND 경쟁 8개+ |
